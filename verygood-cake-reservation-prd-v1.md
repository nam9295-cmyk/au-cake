# Verygood Chocolate Cake Reservation PRD v1

> 베리굿초콜릿의 단일 케이크 예약 사이트 기획서  
> Codex는 이 문서를 기준으로 고객 예약 페이지와 관리자 페이지를 차근차근 구현한다.

---

## 0. 프로젝트 요약

### 프로젝트명

**Verygood Cake Reservation**

### 목적

베리굿초콜릿의 대표 케이크인 **Gâteau au Chocolat / 초코케이크**를 고객이 모바일에서 간편하게 사전 예약할 수 있는 사이트를 만든다.

이 사이트는 일반 쇼핑몰이 아니다.  
**단일 프리미엄 케이크를 고급스럽게 소개하고, 고객이 카카오 농도와 픽업 날짜/시간을 선택해 예약 신청하는 모바일 우선 예약 시스템**이다.

### 핵심 방향

- 상품은 `Gâteau au Chocolat` 1종류만 운영한다.
- 고객 UI는 모바일 우선으로 만든다.
- 복잡한 커스텀 주문 UI를 만들지 않는다.
- 고객은 카카오 농도, 픽업 날짜/시간, 예약자 정보만 입력한다.
- 온라인 결제는 v1에서 구현하지 않는다.
- 예약 신청 후 관리자가 admin에서 확인하고, 고객에게 보낼 안내 문자를 복사해 직접 발송한다.
- 예약 내역은 admin에서 확인하고 엑셀 또는 CSV로 다운로드할 수 있게 한다.
- DB 구조는 단순하게 유지한다.
- Appwrite DB 스키마는 가능하면 수동 생성이 아니라 CLI 또는 setup script로 자동 생성한다.

## 1. 사용자

### 주요 고객

- 20~50대 여성
- 생일, 기념일, 선물용 케이크를 찾는 고객
- 모바일로 예약하는 고객이 대부분일 것으로 예상
- 복잡한 UI보다 빠르고 쉬운 예약 흐름을 선호

### 내부 사용자

- 베리굿초콜릿 운영자
- 제니
- 매장 예약 확인 담당자

내부 사용자는 admin에 로그인해 예약을 확인하고, 상태를 변경하고, 문자 안내 내용을 복사할 수 있어야 한다.

## 2. 상품 정보

### 상품명

- 영문명: `Gâteau au Chocolat`
- 한글명: `베리굿 초코케이크`

### 상품 설명

밀도 높은 초코 시트와 파베초콜릿 가나슈를 4단으로 샌드한 베리굿의 리얼 초콜릿 케이크.

### 상품 특징

- 케이크 1종류
- 1호 사이즈
- 6~8인 추천
- 4단 초코 시트
- 파베초콜릿 가나슈 샌드
- 크림층 없이 진한 초콜릿 밀도를 강조
- 케이크 위 기본 장식: **카카오 57.5% 초콜릿 에펠탑**
- 레터링 없음
- 초 옵션은 v1에서는 구현하지 않음
- 향후 add-on 옵션으로 확장 가능

### 고객 화면에 노출할 핵심 문구

```text
Gâteau au Chocolat
베리굿 초코케이크 예약

밀도 높은 초코 시트와
파베초콜릿 가나슈를 4단으로 샌드한
베리굿의 리얼 초콜릿 케이크.

하루 5개 한정, 사전예약제로 준비됩니다.
```

### 상세 설명 문구

```text
진짜 초코케이크를 찾으셨나요?

베리굿의 가또 쇼콜라는
크림층 없이 초콜릿의 밀도를 가득 담은 4단 케이크입니다.

초코 시트 사이사이에 파베초콜릿 가나슈를 샌드해
한 조각만으로도 깊고 진한 만족감을 느낄 수 있습니다.

케이크 위에는 베리굿에서 직접 만든
카카오 57.5% 초콜릿 에펠탑이 기본으로 올라갑니다.
```

## 3. 카카오 농도 옵션

고객은 가나슈에 사용되는 카카오 농도를 선택할 수 있다.

### 옵션명

고객 화면에서는 `가나슈 카카오 농도 선택`이라고 표시한다.

### 옵션

#### 57.9% Classic

```text
부드럽고 달콤한 기본 밸런스.
아이와 함께 드시거나, 처음 주문하시는 분께 추천합니다.
```

#### 70.5% Dark

```text
덜 달고 진한 초콜릿 풍미.
커피와 함께 먹기 좋은 어른의 초콜릿 맛입니다.
```

#### 100% Cacao

```text
가장 깊고 쌉싸름한 카카오 풍미.
단맛보다 카카오 본연의 향과 여운을 즐기는 분께 추천합니다.
```

### 주의 안내 문구

100% 옵션에 대한 고객 오해를 막기 위해 다음 안내를 표시한다.

```text
카카오 농도는 가나슈에 사용되는 초콜릿 풍미 기준입니다.
케이크 전체의 당류 함량을 의미하지 않습니다.
```

### 가격 정책

- 57.9%, 70.5%, 100% 모두 동일 가격
- v1에서는 농도별 추가금 없음
- 가격은 settings 또는 코드 상수로 관리

## 4. 예약 정책

### 예약 방식

- 고객이 예약 신청을 한다.
- 예약 신청은 즉시 확정이 아니다.
- 관리자가 admin에서 예약 내용을 확인한다.
- 관리자가 고객에게 보낼 안내 문자를 복사한다.
- 운영자가 고객에게 직접 문자 발송한다.
- 입금 확인 후 예약을 최종 확정한다.

### 고객에게 노출할 표현

`예약 완료`보다 `예약 신청 완료`라고 표현한다.

### 예약 완료 화면 안내 문구

```text
예약 신청이 완료되었습니다.

매장에서 예약 가능 여부를 확인한 뒤
확정 안내 문자를 보내드립니다.

입금 확인 후 예약이 최종 확정됩니다.
```

### 결제 방식

- v1에서는 온라인 결제 없음
- 계좌번호 안내 방식
- 계좌번호는 고객 페이지 하단, 예약 완료 화면, 관리자 복사용 문자에 포함

### 하루 한정 문구

고객 화면에는 희소성을 위해 다음 문구를 노출한다.

```text
하루 5개 한정 제작
좋은 컨디션의 케이크를 위해 매일 한정 수량만 예약제로 준비합니다.
```

또는

```text
하루 5개, 사전예약 한정 제작
베리굿의 가또 쇼콜라는 매일 정해진 수량만 정성스럽게 준비됩니다.
```

### 실제 시스템 제한

- 고객 화면에는 하루 5개 한정 문구를 노출한다.
- 시스템에서 자동으로 5개 초과 예약을 차단하지 않는다.
- 예약 가능 여부는 관리자가 최종 확인한다.
- 나중에 필요하면 날짜별 마감 기능 또는 예약 수 제한 기능을 추가한다.

### 픽업 시간

평일:

```text
07:00 - 21:00
```

주말:

```text
09:00 - 22:00
```

> 주말 마감 시간은 10pm 기준으로 `22:00`으로 반영한다.  
> 실제 운영 시간이 오전 10시 마감이라면 settings에서 `10:00`으로 변경할 수 있게 한다.

### 운영시간 외 픽업 안내

```text
운영시간 외 픽업을 원하시면 요청사항에 남겨주세요.
확인 후 연락드리겠습니다.
```

## 9. 관리자 문자 복사 기능

관리자 페이지에 `문자 복사` 버튼을 만든다.

버튼 클릭 시 예약 정보 기반으로 아래 문자를 생성해 클립보드에 복사한다.

### 문자 템플릿

```text
[베리굿초콜릿 케이크 예약 안내]

안녕하세요. 베리굿초콜릿입니다.
예약 신청해주신 가또 쇼콜라 예약 내용 안내드립니다.

예약번호: {{reservationNumber}}
제품명: Gâteau au Chocolat 초코케이크
카카오 농도: {{cacaoPercent}}
픽업일: {{pickupDate}}
픽업시간: {{pickupTime}}
예약자명: {{customerName}}

케이크는 픽업 시간에 맞춰 준비됩니다.
예약 확정을 위해 아래 계좌로 입금 부탁드립니다.

입금계좌: {{bankName}} {{bankAccount}}
예금주: {{accountHolder}}
입금자명은 예약자명과 동일하게 부탁드립니다.

입금 확인 후 예약이 최종 확정됩니다.

주소: {{storeAddress}}
문의: {{storePhone}}

감사합니다.
베리굿초콜릿
```

### 추가 요구사항

- 문자 복사 성공 시 toast 알림 표시
- 예: `문자 내용이 복사되었습니다.`
- 문자 복사 시간이 필요하면 `smsCopiedAt` 필드는 추후 추가
- v1에서는 smsCopiedAt 없이 구현해도 됨

## 11. DB 설계 원칙

### 핵심 원칙

DB를 과도하게 나누지 않는다.

현재 상품은 `Gâteau au Chocolat` 1종류이므로 v1에서는 다음 테이블을 만들지 않는다.

- products
- cacao_options
- add_ons
- pickup_slots

상품 정보, 에펠탑 장식 설명, 레터링 없음 안내, 하루 5개 한정 문구 등은 코드 상수 또는 settings로 관리한다.

### v1 필요 테이블

- reservations
- settings

### 선택 테이블

- blocked_dates는 v1에서는 만들지 않아도 됨
- 나중에 휴무일/마감일 관리가 필요하면 추가

## 12. Appwrite DB 요구사항

### 자동 생성

Appwrite 콘솔에서 컬럼을 하나씩 직접 만들지 않도록 자동 생성 방식을 준비한다.

우선순위:

1. `appwrite.json` + Appwrite CLI push 방식
2. 또는 `scripts/setup-appwrite.ts` 스크립트 방식

Codex는 프로젝트 상황에 맞게 하나를 선택해 구현한다.

### 환경변수 예시

```env
NEXT_PUBLIC_APPWRITE_ENDPOINT=
NEXT_PUBLIC_APPWRITE_PROJECT_ID=
APPWRITE_API_KEY=
APPWRITE_DATABASE_ID=
APPWRITE_RESERVATIONS_TABLE_ID=reservations
APPWRITE_SETTINGS_TABLE_ID=settings
```

클라이언트에 노출되면 안 되는 값은 `NEXT_PUBLIC_`를 붙이지 않는다.

## 15. 권한 / 보안

### 고객 사이트

- 고객은 예약 신청을 생성할 수 있어야 한다.
- 고객이 전체 예약 목록을 읽을 수 있으면 안 된다.
- 예약 조회 기능이 있으면 예약번호와 연락처 확인 후 본인 예약만 보여준다.

### 관리자 사이트

- 로그인 사용자만 접근 가능
- 예약 목록 조회 가능
- 예약 상세 조회 가능
- 예약 상태 변경 가능
- 입금 상태 변경 가능
- 관리자 메모 수정 가능
- 엑셀/CSV 다운로드 가능

### 추천 구조

민감한 작업은 가능하면 서버 API 또는 Appwrite Function을 통해 처리한다.

예:

- 예약 생성
- 예약번호 생성
- 관리자 예약 수정
- 예약 조회
- 문자 템플릿 생성

## 24. 완료 기준

### 고객 사이트

- 모바일에서 제품 소개를 보고 예약 신청까지 완료할 수 있다.

- 예약 상세를 볼 수 있다.
